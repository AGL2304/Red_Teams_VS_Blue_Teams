Défis · Obstacles Rencontrés

Problèmes et Solutions

What went wrong (and how we fixed it)

⚠️
Défi 1: Pipeline d'alerte lent
Problème: Les alertes mettaient 20-30 secondes à s'indexer.
Solution: Créer un systeme d'injection directe Elasticsearch → alertes instantanées.
⚠️
Défi 2: Erreurs XML dans les règles
Problème: Wazuh rejetait les règles avec erreurs d'encoding.
Solution: Valider et corriger les fichiers XML, redémarrer le manager.
⚠️
Défi 3: Faux positifs WAF
Problème: 44% de faux positifs (56% effectiveness).
Solution: Affiner les regex et patterns pour réduire bruit.
✓
Défi 4 (RÉSOLU): Permissions Docker
Problème: Permission denied sur fichiers certificats.
Solution: chmod -R 777 avant suppression, puis cleanup.

Leçons Apprises · Lessons Learned

5 Key Takeaways

Pour future projects

1. Collaboration Red+Blue
La vraie sécurité vient de l'attaque ET la défense ensemble. Red seul = pas de défense. Blue seul = pas de validation.
2. Automatisation > Manual
Les scripts d'injection d'alertes étaient critique pour quick testing. CI/CD speeds up iteration.
3. Monitoring time-sensitive
20-30 seconds latency = too slow for incident response. Direct Elasticsearch polling = instant alerts.
4. Documentation crucial
Без docs, ничего не работает. Tout doit être écrit: runbooks, règles, procedures.
5. Iterative improvement
56% effectiveness ≠ perfection. Mais c'est mieux que 0%. Improve progressively, not perfect at once.
